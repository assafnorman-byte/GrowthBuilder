01 / HOW I OPERATE

I optimize on signal, not on dashboards.

Most teams over-trust platform optimization and declare a channel a "winner" off attribution it can't prove. I run a three-tier signal model that decides what earns budget — and what only earns a watch.

The hierarchy is the whole point: what you trust determines what you scale.

TIER 1
Decide

SQL & MQL — volume + value

The only signals strong enough to allocate budget against. Optimization targets, bid feeds and scaling calls anchor here.

→ Drives budget & scaling decisions.
TIER 2
Diagnose

Platform conversions · CTR / CPC / CVR

Useful for diagnosing what's moving and where to test — never the basis for a winner call on its own.

→ Informs tests, not budget.
TIER 3
Context

Attribution

Weak and non-deterministic — correlation, not causation. Read it for direction; validate real lift with incrementality tests.

→ Read for direction only.
"Trust the signal enough to optimize — not enough to blindly scale."

// high-intent SQL → server-side conversion
const sql = $json.body.lead;
const payload = {
  event_name: "sql_qualified",
  event_time: Math.floor(Date.now()/1000),
  user_data: {
    em: sql.email.hash(),          // sha256
    external_id: sql.crm_id.hash(),
    client_user_agent: sql.ua      // EMQ boost
  },
  custom_data: { value: sql.pltv, currency: "USD" }
};
return postTo("meta.capi", payload);
